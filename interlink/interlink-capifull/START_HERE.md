## 🎯 What MCPAPI v2.0 Does

### Core Capability
Provides **governed, audited, intelligent machine-to-machine capability execution** with automatic breach prevention.

### Every Request Gets
- ✅ **Verified** — Signature validation + replay prevention
- ✅ **Authorized** — Policy composition + approval quorum
- ✅ **Observed** — Behavioral anomaly detection
- ✅ **Executed** — With error handling
- ✅ **Proven** — PGL-backed immutable evidence
- ✅ **Audited** — Complete compliance trail
- ✅ **Learned** — Patterns extracted for prediction

### The 3 Layers

**Safety Layer (Breach Prevention)**
- Anomaly detection (behavioral spikes)
- Request quarantine (hold suspicious requests)
- Approval quorum (M-of-N signatures)
- Replay prevention (nonce-based)
- Trust suppression (automatic on critical anomalies)

**Intelligence Layer (Learning & Prediction)**
- Cost attribution (per-agent, per-capability)
- Behavioral learning (baseline → detect deviations)
- Risk scoring (0-100 threat level)
- Anomaly correlation (link signals → detect attacks)

**Governance Layer (Policy Composition)**
- System + owner + runtime policies merged
- Temporal adjustments (business hours vs after-hours)
- Delegation chains (track lineage, limit depth)
- Effective permissions (what can agent do NOW?)

## 🏗️ Architecture (9 Phases)

```
Request → Phase 1: Identity & Security (15ms)
        → Phase 2: Capability & Policy (40ms)
        → Phase 3: Safety & Anomaly (20ms)
        → Phase 4: Cost & Budget (5ms)
        → Phase 5: Approval Workflows (2ms)
        → Phase 6: Execution (100-5000ms)
        → Phase 7: Evidence & Proof (50ms)
        → Phase 8: Audit & Compliance (10ms)
        → Phase 9: Response (5ms)
        ↓
    Response (with evidence_hash + metadata)
```

**Total Latency (excl. capability):** ~150ms  
**Throughput:** 1,000-50,000 req/s (depends on scale)
