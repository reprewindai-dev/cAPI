# Veklom Trust Connection Contracts

This directory is the canonical, language-neutral source of truth for the
Veklom governed unified-call system. It is owned by the cAPI repository.

- `schemas/v1/` — versioned JSON Schemas for the Trust Connection protocol
  (`TrustConnectionV1`, `ConnectionOperationV1`, `OperationResultV1`, receipt
  and reference types). Regenerated by `scripts/emit-schemas.py`; checked-in
  output must match the script.
- `graphql/schema.graphql` — the capability-centric GraphQL SDL served by
  veklom-byos-backend at `POST /api/v1/graphql` and consumed by SDK codegen
  in `reprewindai-dev/veklom-sdk`.
- `graphql/operations/` — the persisted, registered GraphQL operations that
  external machine clients may execute. Arbitrary GraphQL strings are rejected
  in production.
- `graphql/operation-registry.json` — generated by
  `scripts/build-operation-registry.py`. Maps each operation to its capability
  ID, lane, version, sha256 operation hash, and depth/cost limits.

## Execution lanes

| Lane | Meaning | Path |
|---|---|---|
| 1 | Direct governed read | persisted query → ConnectionContext → transaction-local identity → PostgreSQL RLS |
| 2 | Bounded deterministic write | persisted mutation → constrained DB function → nonce/idempotency check → evidence outbox |
| 3 | Consequential side effect | invokeCapability → Interlink cAPI → CAPPO authorization → execution → PGL evidence → VNP measurement → x402 when applicable |

No Lane 3 execution may bypass CAPPO.

## Compatibility rules

1. Minor versions may add types, fields, enum values, and operations.
2. Minor versions may never remove or retype existing fields.
3. An operation hash change requires an operation version bump.
4. Breaking schema changes require a new major `schema_version`; connections
   pin a schema version and reject mismatches.
5. Money is always integer minor units. No floats.

## CI gates

`python3 contracts/scripts/build-operation-registry.py --check` must pass:
the checked-in registry must match the operations, every operation must carry
a capability ID and lane, and hashes must be current.
